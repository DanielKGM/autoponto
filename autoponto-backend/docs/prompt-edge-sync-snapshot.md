# Prompt Edge Sync Snapshot

Atualize `referencia-edge/autoponto-edgenode/services/edge-app/app/sync.py` para consumir apenas o snapshot Redis da API principal: `executar_sincronizacao` deve chamar `GET {AUTOPONTO_API_URL}/edge/pull/` apenas com `node_id`, receber `{"snapshot_data":"YYYY-MM-DD","synced_at":"ISO-8601","cache_redis":{...}}`, validar esses tres campos, chamar `substituir_snapshot_redis(cache_redis, snapshot_data, synced_at)` e depois enviar presencas pendentes quando habilitado; nao envie `full=true`, nao envie `cursors`, nao processe `deleted`, nao mantenha `SyncState` nem qualquer cache SQLite replicado de `Sala`, `Dispositivo`, `Aula`, `Aluno`, `MatriculaTurma` ou `EmbeddingFacial`, mantenha a CLI apenas com `--sem-presencas`, e preserve a regra operacional de sincronizar ao iniciar, na virada do dia e antes de cada aula salva no snapshot, pois o backend agora considera o pull um snapshot autoritativo pronto para Redis e nao expoe entidades completas nem sincronizacao incremental.
