# Prompt Edge Sync Snapshot

Atualize `referencia-edge/autoponto-edgenode/services/edge-app/app/sync.py` para abandonar completamente o contrato incremental antigo: remova `msgpack`, `SyncState`, `_cursores_atuais`, `_salvar_cursores`, `_aplicar_exclusoes`, o parametro `forcar_completa`, o envio de `full=true`, o envio de `cursors` e qualquer processamento de `deleted`; `executar_sincronizacao` deve sempre chamar `GET {AUTOPONTO_API_URL}/edge/pull/` apenas com `node_id`, receber um snapshot no formato `{"data": {...}, "synced_at": "..."}`, limpar as tabelas locais replicadas (`EmbeddingFacial`, `MatriculaTurma`, `Aula`, `Dispositivo`, `Aluno`, `Sala`) antes de aplicar o payload, inserir/upsertar todos os itens de `data`, reconstruir o Redis ao final e depois enviar presencas pendentes quando habilitado; ajuste a CLI removendo `--completa`, mantendo apenas `--sem-presencas`, e preserve a regra operacional de sincronizar ao iniciar, na virada do dia e antes de cada aula salva no cache, pois o backend agora considera o pull um snapshot autoritativo do dia e nao expõe mais cursores, incremental, `deleted` ou auditoria de sincronizacao.
